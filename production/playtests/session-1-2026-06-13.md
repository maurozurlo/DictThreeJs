# Playtest Session 1 — New Player Experience
**Date**: 2026-06-13
**Tester**: Mauro Zurlo (developer)
**Difficulty**: Not specified (default)
**Run outcome**: Completed multiple rounds; observed HUD, tutorial, and mid-game systems

---

## Focus: Core loop clarity within round 1

**Finding**: Tutorial is present and effective. Game is understandable on first contact.

**Nit — Difficulty selection UI**: When clicking New Game, three difficulty buttons appear. They use the same button style as all other UI buttons — no visual hierarchy or grouping. Feels out of place. Options explored: smaller buttons, modal/dialog, same row layout, margined/framed group.

**Idea — Player name input**: Optional text input on the New Game / difficulty screen. If left blank, defaults to "The Leader" (EN) / "Él Líder" (ES). Name can be referenced later: end-game screen, statue plaques, laws, deals.

---

## Focus: Coup warning legibility

**Finding — BUG**: Coup warning only surfaces in the DayEnded dialog. Player had low charisma and a faction near danger zone with no advance signal. By the time DayEnded showed the warning, there was no round left to adjust. Feels unfair.

**Expected**: Player should receive at least one round of visible advance warning before a coup fires — either in the log at round end, or a persistent indicator on the HUD while the condition is active. The `coupWarningFaction` and `coupArmedLastRound` state already exists; it is not being surfaced prominently enough.

---

## Focus: Advance round button (>)

**Finding**: Button works. Mild confusion about when it's available. Suggestion: animated tooltip or visual indicator when all required actions for the round are complete (law decided + deal decided + meet action taken).

---

## Focus: Grace period (Story 4-5)

**Finding**: Round 1 is noticeably more forgiving even if the player can't articulate why. Relation losses feel manageable. System is working as intended. Timer not obviously longer but not a problem.

---

## Additional observations

**Daily events (round-start random relation adjustments)**: Feels unnecessary and slightly unfair. The game already has enough complexity from periodic events, budget effects, and player decisions. Random relation deltas at round start reduce player agency without adding interesting decisions. Candidate for removal.

**Treasury deficit from periodic events**: Agreeing to a costly periodic event (e.g. lavish meeting for $100M) when treasury is $50M results in negative treasury. Game allows this — player survives the round in deficit and must recover before advancing. Discovered to be fun gameplay, not a breaking bug. Inspires potential feature: sell back purchased shop items (statues, media coverage) at a reduced price as an emergency option.
